Funzionamento interfaccia web

Il Pannello operatore è dotato di un web server che consente di accedere con il browser ad una serie di pagine WEB tramite le quali è possibile verificare alcune impostazioni del pannello o effettuare operazioni di upload e download.

L’indirizzo IP impostato nel pannello è visualizzabile nell’HMI accedendo alla pagina info raggiungibile dal menu principale dell’applicazione di dimostrazione.

Per utilizzare l’interfaccia web occorre digitare l’indirizzo IP del pannello nella barra della URL del browser utilizzato.

Per l’accesso all’interfaccia WEB viene richiesto l’inserimento di una password. Le password determinano i permessi di accesso alle funzionalità dell’interfaccia web. I valori di default sono così impostati:

  • User : 111111

  • SuperUser: 222222

  • Admin: 1357924680

  • LAN Config

    In questa pagina è possibile impostare le proprietà di rete del pannello. Il pannello supporta solo una configurazione di rete statica per la quale è possibile definire l’indirizzo IP, la maschera di rete (NM), il default gateway (GW) e i server dns (DNS1 e DNS2)

  • Date and Time

    La pagina “Date and Time” rileva automaticamente data e ora del pc su cui viene eseguito il browser che si connette al pannello e permette di impostare i valori rilevati sul pannello stesso.

  • Display Settings

    La pagina Display Settings permette di modificare le seguenti caratteristiche:

    • Luminosità del display: è possibile settare un valore di luminosità tra 0 e 100 corrispondenti rispettivamente alla minima e alla massima luminosità ammessa

    • Screensaver: è possibile impostare il tempo di inattività del display trascorso il quale si attiva lo screensaver di sistema. Il valore impostato deve essere espresso in secondi. Un valore pari a 0 disattiva lo screensaver.

    • Touch Calibration: è possibile avviare la procedura di calibrazione del touchscreen associato al display. La procedura di calibrazione interrompe l’esecuzione del software HMI e PLC normalmente attivo sul pannello, e richiede all’utente di toccare i punti evidenziati sul display. Al termine della calibrazione l’esecuzione del software HMI e PLC riprende automaticamente.

  • Manage Configuration

    Permette all’utente di effettuare l’aggiornamento della splash screen (splash.png) che può essere visualizzato alla partenza dell’applicazione HMI

  • Alarms

    La pagina Alarms avvia la componente dell’interfaccia web dedicata alla gestione degli allarmi.
    Sono previste due tipologie di operazioni effettuabili sugli allarmi: il download e la cancellazione dei log presenti sul pannello.

  • Logs

    La pagina Logs avvia la componente dell’interfaccia web dedicata alla gestione degli storici. Sono previste due tipologie di operazioni effettuabili sugli storici: il download e la cancellazione degli storici.

  • Trend

    La pagina Trend avvia la porzione di interfaccia dedicata alla gestione dei trend. Sono previste tre tipologie di operazioni effettuabili sui trend: l’upload il download e la cancellazione.

    Upload:

    La funzione di upload di famiglie di trend aggiorna l’elenco delle famiglie di trend disponibili sul pannello. Le famiglie di trend da caricare devono essere compresse in un archivio in formato zip . L’archivio è composto da una lista di files ognuno dei quali descrive una tipologia di trend.

  • Recipes

    La pagina Recipes avvia la porzione di interfaccia dedicata alla gestione delle ricette.

    Sono previste tre tipologie di operazioni effettuabili sulle ricette: l’upload il download e la cancellazione.

    Upload:

    La funzione di upload delle ricette aggiorna l’elenco delle ricette disponibili sul pannello. Le ricette da caricare devono essere compresse in un archivio in formato zip. L’archivio è organizzato in cartelle ciascuna delle quali corrisponde ad una tipologia di ricetta e, all’interno di ciascuna cartella, si trovano le varie implementazioni di un tipo di ricetta.

    Ad esempio se si ipotizza di avere due famiglie di ricette una per il raffreddamento e l’altra per il riscaldamento all’interno dell’archivio contenente le ricette si avrà la seguente l’organizzazione:

    ├── cool

    │ ├── cool_10

    │ ├── cool_20

    │ └── cool_30



    └── hot

    ├── hot_100

    ├── hot_150

    ├── hot_200

    └── hot_50

  • Screenshot

    La pagina Screenshot avvia la componente dell’interfaccia web dedicata alla gestione degli Screenshots.
    Sono previste due tipologie di operazioni effettuabili sugli Screenshot: il download e la cancellazione dei Screenshots presenti sul pannello.

  • Manage Password

    Permette all’utente che ha effettuato il login sull’interfaccia con password “User” o “Super User” di modificare unicamente la propria password. La password associata al livello “Admin” non è modificabile. NOTA: queste password, sono diverse dalle password usate nella applicazione HMI.
