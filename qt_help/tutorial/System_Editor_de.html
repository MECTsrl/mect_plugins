System Editor

Mit dem System Editor ist es möglich, einige Konfigurationen für die grafische Oberfläche und die Kommunikationsparameter zwischen dem Panel und den entfernten Knoten zu ändern.

Um den “System Editor” zu öffnen, wählen Sie das MectSuite-Symbol:


Hier ist das “System Editor”:

Im oberen Teil der Registerkarte “System” befindet sich die Schaltfläche zum Speichern der Einstellungen. Unmittelbar darunter befinden sich einige Registerkarten: System, SERIAL PORT 0 usw…,die je nach den verfügbaren Ports des Produkts bearbeitet werden können oder nicht.

Hier sind Eigenschaften des “System Editor”:

  • System: Alle allgemeinen Eigenschaften des Systems sind eingestellt:

    • Pwd Logout Page: Seite, die angezeigt wird, wenn die Sitzung abläuft. Wenn das Feld leer ist, bleibt das aktuelle Bildschirm erhalten.

    • Pwd Timeout [s]:Gibt an, nach wie vielen Sekunden der Inaktivität die Sitzung abläuft. Mit dem Wert “0” läuft die Sitzung nie ab.

    • Screen Saver [s]: Zeigt an, nach wie vielen Sekunden der Modus “Screen Saver” in Betrieb genommen wird. Mit dem Wert “0” wird die Einstellung deaktiviert.

    • Slow Log Period [s]: Gibt an, wie viele Sekunden Sie Variablen vom Typ “S” (definiert im Feld “Update” der “Crosstable Editor”).

    • Fast Log Period [s]: Gibt an, wie viele Sekunden Sie Variablen vom Typ “F” speichern möchten (definiert im Feld “Update” deR “Crosstable Editor”). Die Zahl muss kleiner sein als das Feld “Slow Log Period”.

    • Max Log Space [Mb]: Speicherplatz für das Speichern historischer Variablen. Variablen.

    • Trend Window [s]: Enthält den Wert des Zeitfensters der Trendanzeige visualizzazione dei trend. Die Zahl muss mindestens das Dreifache des Feldes “Fast Log Period” betragen.

    • Retries: Zeigt die Anzahl der Fehler an, die einem Gerät erlaubt sind, bevor die Abfragequote reduziert wird.

    • Blacklist: : Reduzierte Abfragequote, wenn die im Feld “Retries” eingestellte Anzahl überschritten wird. .

    • Start Page: Name der Seite für den Bildschirm, den Sie zuerst anzeigen möchten. Im Qt Creator muss eine Seite mit diesem Namen vorhanden sein. Es werden nur die Seiten angezeigt, die beim Öffnen des Projekts vorhanden sind.

    • Home Page: : Name der Seite, die Sie der “Home Page” zuordnen möchten. Im Qt Creator muss eine Seite mit diesem Namen vorhanden sein. Es werden nur die Seiten angezeigt, die beim Öffnen des Projekts vorhanden sind.

    • Buzzer Touch: Zeigt an, ob jedem Druckereignis auf dem Touchscreen ein akustisches Signal zugeordnet werden soll.

    • Buzzer Alarm: Zeigt an, ob das Auftreten einer Alarmbedingung mit einem akustischen Signal verbunden werden soll.

    • Read Period 1 [ms]: gibt an, wie oft die Variablen mit dem Wert “1” im Feld “Priority” der Crosstable gelesen werden.

    • Read Period 2 [ms]: gibt an, wie oft die Variablen mit dem Wert “2” im Feld “Priority” der Crosstable gelesen werden.

    • Read Period 3 [ms]: gibt an, wie oft die Variablen mit dem Wert “3” im Feld “Priority” der Crosstable gelesen werden.

      Das Feld “Priority” wird bei Variablen mit PLC-Protokoll nicht berücksichtigt.

  • SERIAL PORT 0 / 1 / 2 / 3: Alle Kommunikationsparameter des seriellen Gerätes sind eingestellt.

    Im Feld “Max Block Size” können Sie die maximale Anzahl von Variablen innerhalb eines Blocks festlegen.

    ATTENZIONE: Die Neuordnung der Blöcke erfolgt nach dem Speichern der Crosstable. Die internen seriellen Parameter dürfen nicht verändert werden, da sonst das Gerät nicht richtig funktioniert.

  • TCP IP PORT: Sie stellen die Parameter für die TCP IP-Kommunikation mit einem entfernten Gerät ein:

    • Timeout [ms]: maximale Wartezeit, bis der Master eine Antwort vom Slave erhält.

    • Silence [ms]: Häufigkeit der aufeinanderfolgenden Anfragen von einem Master an einen Slave.

    • Max Block Size [1…64]: maximale Anzahl von Variablen innerhalb eines Blocks. ACHTUNG: Die Neuordnung der Blöcke erfolgt nach dem Speichern der Crosstable.

  • CANOPEN 0 / 1: Einstellungen der Baudrate des CANopen-Protokolls.
