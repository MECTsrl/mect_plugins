Funcionamiento de las contraseñas

Hay dos formas diferentes de introducir las contraseñas:

  • Asociar una variable en la propiedad PasswordVar de un ATCMbutton o introducir una constante directamente para proteger:

    • un cambio de página

    • la acción del botón.

  • En cualquier página creada, excepto las de la biblioteca, es posible asociar contraseñas del sistema para permitir el acceso.

    El mecanismo de la contraseña se gestiona de forma jerárquica y existen los siguientes niveles:

    • Admin

    • SuperUser

    • User

    • Operator

    El nivel “Admin” es la prioridad más alta y “User” la más baja, mientras que “Operator” indica que no se ha realizado ningún login, por lo que no hay ninguna contraseña activa (operación por defecto).

    Las contraseñas son estrictamente numéricas sin ninguna limitación particular en el número de dígitos.

    En la aplicación del panel de control hay dos claves que intervienen en la gestión de las contraseñas:

    • tecla de inicio de sesión en la página del menú: le permite iniciar sesión en el sistema o establecer el nivel de acceso actual. Pulsando la tecla aparece el teclado numérico en el que se debe introducir la contraseña. A partir de este momento las operaciones en la interfaz gráfica están condicionadas al nivel de acceso del usuario que se autentica. El botón de inicio de sesión se convierte en un botón de cierre de sesión. Para cerrar la sesión, simplemente presione el nuevo botón de cierre de sesión.

    • clave de contraseña presente en la página de options: permite modificar la contraseña del usuario conectado actualmente. Pulsando la tecla aparece el teclado numérico en el que se debe introducir la nueva contraseña que se asociará al nivel de usuario que previamente ha accedido al sistema.

    Para configurar y proteger cualquier página de la contraseña debe abrir la página correspondiente .cpp.

    Para proteger la página es suficiente al principio del generador de páginas (page”N”::page”N”(QWidget *parent) ), justo debajo de la línea:

    ui->setupUi(this);

    añadir la línea:

    protection_level = nivel mínimo de acceso;

    donde corresponde el “nivel mínimo de acceso” a:

    • pwd_admin_e
    • pwd_super_user_e
    • pwd_user_e
    • pwd_operator_e

    Por ejemplo, si pongo el nivel protection_level en pwd_user_e, puedo entrar en la página sólo si estoy conectado con la contraseña de “User”, “SuperUser” o di “Admin”. De lo contrario, aparecerá una ventana emergente para notificarle que no se le permite acceder a la página. Recuerde guardar los cambios en la página.

    Si no se especifica ningún nivel de acceso en el constructor de página, la página tiene pwd_operator_e como nivel de acceso predeterminado, es decir, la página es accesible para todos los niveles de usuario proporcionados y también para cualquier usuario que no haya iniciado sesión en el sistema.

    NOTA: Cada usuario sólo puede cambiar su propia contraseña. Es decir, el usuario “User” sólo puede cambiar su contraseña y el usuario “Admin” no puede cambiar la contraseña de “User”.

    La clave de contraseña se desactiva hasta que el usuario inicia sesión.

    El inicio de sesión no caduca hasta que se reinicia el panel o si no ha cerrado la sesión de forma explícita pulsando el botón correspondiente.
