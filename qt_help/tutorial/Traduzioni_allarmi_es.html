Traducción de las alarmas

El comentario de alarma no puede ser visto/traducido por Qt Linguist porque no es parte de un elemento gráfico sino que está dentro del Crosstable Editor.

Sin embargo, es posible hacer una traducción "manual" de las alarmas introduciendo el código en la pagexxx.cpp (página .cpp donde se hace el cambio de idioma).

Ver el proyecto "Alarm_translations" en C:\Qt485\MectApps\Examples.

ATENCIÓN: las alarmas en el Crosstable Editor deben definirse con caracteres ASCII.

Procedimiento:

  • Copie la siguiente función en la parte inferior de pagexxx.cpp:

    void setEventDescription(QString name, QString description)
    {
         event_t * event = EventHash.find(name).value();
         if (event) {
            strncpy(event->description, description.toLatin1().data(), DESCR_LEN);
         }
    }

  • En la correspondiente pagexxx.h, copie dentro de "private slots":

    void changeEvent(QEvent * event);

  • Llame a la función setEventDescription() introduciendo la traducción que quiera.

    Si utiliza un botón ATCM para cambiar el idioma, debe llamar la función setEventDescription(Nombre de la variable de alarma, Texto a mostrar) en la llamada del botón (Go to slot...), véase la opción "2" en el capítulo “Seleccionar el idioma desde el dispositivo”.

    Por ejemplo: setEventDescription("Alarma_temp", "Alarma de temperatura - Entrada analógica 1");

    Sigue este último paso para todas las alarmas de lenguaje.
