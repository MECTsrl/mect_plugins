System Editor

Con il System Editor è possibile modificare alcune configurazioni per l’interfaccia grafica e i parametri di comunicazione tra il pannello e i nodi remoti.

Per avviare il “Crosstable Editor” vedere la figura seguente:


Ecco come si presenta il “System Editor”:

Nella parte alta dell’editor sono presenti alcuni Tab: System, SERIAL PORT 0 etc…., mentre nella parte inferiore è presente il tasto “SAVE” (salvataggio delle impostazioni). In funzione del prodotto che si utilizza si hanno dei parametri di default.

Ecco le caratteristiche del “System Editor”:

  • System: si impostano tutte le caratteristiche generali del sistema:

    • Pwd Logout Page: Pagina visualizzata alla scadenza della sessione. Se il campo è vuoto si rimane nella schermata attuale.

    • Pwd Timeout [s]: Indica dopo quanti secondi di inattività scade la sessione. Con il valore “0” la sessione non scade mai.

    • Screen Saver [s]: Indica dopo quanti secondi entra in funzione la modalità di “Screen Saver”. Con il valore “0” si ha l’impostazione disabilitata.

    • Slow Log Period [s]: Indica ogni quanti secondi si desidera memorizzare le variabili di tipo “S” (definita nel campo “Update” del “Crosstable Editor”).

    • Fast Log Period [s]: Indica ogni quanti secondi si desidera memorizzare le variabili di tipo “F” (definita nel campo “Update” del “Crosstable Editor”). Il numero deve essere minore rispetto al campo “Slow Log Period”.

    • Max Log Space [Mb]: Quantità di spazio dedicato al salvataggio delle variabili storicizzate.

    • Trend Window [s]: Contiene il valore della finestra temporale di visualizzazione dei trend. Il numero deve essere almeno il triplo del campo “Fast Log Period”.

    • Retries: indica il numero di errori permessi ad un dispositivo prima di ridurre il tasso di interrogazione.

    • Blacklist: Tasso di interrogazione ridotto nel caso in cui si superi il numero impostato nel campo “Retries”.

    • Start Page: Nome della pagina relativa alla schermata che si vuole visualizzare per prima. In Qt Creator deve essere presente una pagina con questo nome.

    • Home Page: Nome della pagina che si vuole associare alla “Home Page”. In Qt Creator deve essere presente una pagina con questo nome.

    • Buzzer Touch: Indica se ad ogni evento di pressione del touch screen debba essere associata una segnalazione acustica .

    • Buzzer Alarm: Indica se l’insorgere di una condizione di allarme debba essere associata ad una segnalazione acustica.

    • Read Period 1 [ms]: indica ogni quanto tempo sono lette le variabili. Associazione all’elemento “1” nel campo “Priority” della Crosstable.

    • Read Period 2 [ms]: indica ogni quanto tempo sono lette le variabili. Associazione all’elemento “2” nel campo “Priority” della Crosstable.

    • Read Period 3 [ms]: indica ogni quanto tempo sono lette le variabili. Associazione all’elemento “3” nel campo “Priority” della Crosstable.

  • SERIAL PORT 0 / 1 / 2 / 3: si impostano tutti i parametri di comunicazione delle seriali del dispositivo.

    ATTENZIONE: i parametri delle seriali interne non devono essere modificati, pena il malfunzionamento del dispositivo.

  • TC IP PORT: si impostano i parametri per la comunicazione TCP IP con un dispositivo remoto:

    • Timeout [ms]: tempo massimo di attesa che il Master attende per ricevere la risposta dallo Slave.

    • Silence [ms]: frequenza di richieste consecutive da un Master verso uno Slave.

  • CANOPEN 0 / 1: impostazioni del baudrate del protocollo CANopen.

    ATTENZIONE: il Baudrate del protocollo interno non deve essere modificato, pena il malfunzionamento del dispositivo.
