Library pages

In the project there are some default pages ready to use: to do this write page name in the “pageName” property of the ATCMbutton and in particular:

  • “display_settings” window allows to manage the brightness and the setting of the timeout screensaver.

  • “info” window implements the visualization of the board information and of the software loaded.

  • “menu” and “options” windows to realize the basic pages from which show other pages.

  • “recipe_select” window opens the default page for recipes. Recipe is a setup in the field devices of a set of predefined values parameters. The recipe definition is made by filling a file as shown in the paragraph Recipes functioning.

  • “time_set” window sets/modifies date and time.Time set

  • “trend” files implement the management and the visualization of the data stored graphs. See the “Trends” section.

  • “store” alarms display history data (if defined). In the Crosstable it is possible to define variables as history elements. This definition allows the generation of a daily log with saved data. It is possible to define one or more subset of the history data. See the “History functioning” section.

  • “alarms” window implements the visualization of the alarms. See the “Alarm functioning” section.

  • “alarms_history” window implements the visualization of the alarms stored.

  • “data_manager” window implements a menu page to access alarms, logs, trends, recipes, communication pages.

  • “net_conf” window implements the net configuration: Ethernet, Wi-Fi and Mobile.

  • “system_ini” page shows all the panel configuration, graphic behaviour, home page definition, bus configuration. Restart the device after each change.

  • “login_pwd” window displays the login / logout and change password page. See the “Password function” section.

  • “set_password” window displays the page for setting the passwords for the various levels. See the “Password function” section.
