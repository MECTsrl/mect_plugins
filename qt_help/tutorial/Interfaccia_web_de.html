Funktionsweise der Webschnittstelle

Das Bedienfeld ist mit einem Webserver ausgestattet, der den Zugriff mit dem Browser auf eine Reihe von WEBseiten ermöglicht, über die Sie einige Panel-Einstellungen überprüfen oder Upload- und Download-Vorgänge durchführen können.

Die im Panel eingestellte IP-Adresse kann im HMI über die Info-Seite eingesehen werden, die über das Hauptmenü der Demo-Anwendung erreichbar ist.

Um die Webschnittstelle zu verwenden, geben Sie die IP-Adresse des Panels in die URL-Leiste des von Ihnen verwendeten Browsers ein.

Für den Zugriff auf die WEB-Schnittstelle ist ein Passwort erforderlich. Passwörter bestimmen die Zugriffsberechtigungen auf die Funktionalitäten der Webschnittstelle. Die Standardwerte sind wie folgt eingestellt

  • User : 111111

  • SuperUser: 222222

  • Admin: 1357924680

  • LAN Config

    Auf dieser Seite können Sie die Netzwerkeigenschaften des Panels einstellen. Das Panel unterstützt nur eine statische Netzwerkkonfiguration, für die Sie die IP-Adresse, die Netzwerkmaske (NM), das Standard-Gateway (GW) und die DNS-Server (DNS1 und DNS2) definieren können.

  • Date and Time

    Die Seite “Date and Time” erkennt automatisch das Datum und die Uhrzeit des PCs, auf dem der Browser läuft, der sich mit dem Panel verbindet, und ermöglicht es Ihnen, die auf dem Panel selbst erfassten Werte einzustellen .

    Display Settings

    Auf der Seite Display Settings können Sie die folgenden Funktionen ändern:

    • Helligkeit der Anzeige: Es ist möglich, einen Helligkeitswert zwischen 0 und 100 entsprechend der minimal und maximal zulässigen Helligkeit einzustellen.

    • Bildschirmschoner: Sie können die Ausfallzeit der Anzeige einstellen, nach der Bildschirmschoner des Systems aktiviert wird. Der Sollwert muss in Sekunden angegeben werden. Ein Wert von 0 deaktiviert den Bildschirmschoner.

    • Touch Calibration: Sie können den Vorgang zum Kalibrieren des dem Display zugeordneten Touchscreens starten. Die Kalibrierung unterbricht die Ausführung der normalerweise auf dem Bedienfeld aktiven HMI-und SPS-Software und erfordert, dass der Benutzer die auf dem Display markierten Punkte berührt. Nach Abschluss der Kalibrierung startet die HMI- und SPS-Software automatisch wieder.

  • Manage Configuration

    Ermöglicht es dem Benutzer, den Splash-Screen (splash.png) zu aktualisieren, der beim Start der HMI-Anwendung angezeigt werden kann.

  • Alarms

    Die Seite Alarme startet die Web-Interface-Komponente für das Alarmmanagement.
    Es gibt zwei Arten von Operationen, die für die Alarme durchgeführt werden können: Herunterladen und Löschen der Protokolle auf dem Bedienfeld.

  • Logs

    Die Seite Logs startet die Komponente der Webschnittstelle, die für die Verwaltung von Historien vorgesehen ist. Es gibt drei Arten von Operationen, die an Historien durchgeführt werden können: die Aktualisierung der Definitionen von Teilhistorien, das Herunterladen und Löschen von Gesamthistorien.

    Update LOG

    Die Update-Funktion für die Definitionen der Teilhistorie aktualisiert die Liste der verfügbaren Teilhistorien auf dem Panel. Die Definitionen der hochzuladenden Historie müssen Dateien im CSV-Format sein und die Datei muss mit dem Präfix “store” beginnen:

    Datum

    Uhrzeit

    VARIABLE 1

    …

    VARIABLE N

    Die Schnittstelle lädt jeweils eine CSV-Datei hoch, so dass Sie zum Hochladen mehrerer Definitionen mehrere Updates durchführen müssen.

  • Trend

    Die Trend-Seite startet den Interface-Bereich, der dem Trendmanagement gewidmet ist. Es gibt drei Arten von Operationen, die mit Trends durchgeführt werden können: das Upload, der download und das Löschen.

    Upload:

    Die Aktualisierungsfunktion der Trendfamilie aktualisiert die Liste der auf dem Panel verfügbaren Trendfamilien. Die hochzuladenden Trendfamilien müssen in einem Archiv im Zip-Format komprimiert werden. Das Archiv besteht aus einer Liste von Dateien, die jeweils eine Art Trend beschreiben.

  • Recipes

    Auf der Seite Rezepte wird der Schnittstellenbereich für die Verwaltung von Rezepten gestartet.

    Es gibt drei Arten von Operationen, die an den Rezepten durchgeführt werden können: das Upload, das Herunterladen und Löschen.

    Upload:

    Die Funktion zum Aktualisieren von Rezepten aktualisiert die Liste der auf dem Bedienfeld verfügbaren Rezepte. Die hochzuladenden Rezepte müssen in einem Archiv im Zip-Format komprimiert werden. Das Archiv ist in Ordnern organisiert, von denen jeder einer Art von Rezept entspricht, und innerhalb jedes Ordners gibt es die verschiedenen Implementierungen einer Art von Rezept.

    Wenn wir zum Beispiel davon ausgehen, dass wir zwei Familien von Rezepten haben, eine zum Kühlen und eine zum Heizen innerhalb des Archivs, das die Rezepte enthält, werden wir die folgende Organisation haben:

    ├── cool

    │ ├── cool_10

    │ ├── cool_20

    │ └── cool_30



    └── hot

    ├── hot_100

    ├── hot_150

    ├── hot_200

    └── hot_50

  • Screenshot

    Die Seite Screenshot startet die Web-Interface-Komponente für das Screenshot management.
    Es gibt zwei Arten von Operationen, die für die Screenshots durchgeführt werden können: Herunterladen und Löschen der Screenshots auf dem Bedienfeld.

  • Manage Password

    PErmöglicht es dem Benutzer, der sich mit dem Passwort “User” oder “Super User” an der Benutzeroberfläche angemeldet hat, nur sein Passwort zu ändern. Das der Ebene “Admin” zugeordnete Passwort kann nicht geändert werden. HINWEIS: Diese Passwörter unterscheiden sich von den in der HMI-Anwendung verwendeten Passwörtern.
