Übersetzung von Alarmen

Der Alarmkommentar kann von Qt Linguist nicht gesehen/übersetzt werden, da er nicht Teil eines grafischen Elements ist, sondern sich innerhalb des Crosstable-Editors befindet.

Es ist jedoch möglich, eine "manuelle" Übersetzung der Alarme vorzunehmen, indem ein Code in die pagexxx.cpp (Seite .cpp, wo die Sprachänderung vorgenommen wird) eingegeben wird.

Siehe das Projekt "Alarm_translations" unter C:\Qt485\MectApps\Examples.

ACHTUNG: Die Alarme im Crosstable-Editor müssen mit ASCII-Zeichen definiert werden.

Verfahren:

  • Kopieren Sie die folgende Funktion an das Ende von pagexxx.cpp:

    void setEventDescription(QString name, QString description)
    {
         event_t * event = EventHash.find(name).value();
         if (event) {
            strncpy(event->description, description.toLatin1().data(), DESCR_LEN);
         }
    }

  • Kopieren Sie auf der entsprechenden pagexxx.h in "private slots":

    void changeEvent(QEvent * event);

  • Rufen Sie die Funktion setEventDescription() auf, indem Sie die gewünschte Übersetzung eingeben.

    Sie einen ATCM-Button verwenden, um die Sprache zu ändern, müssen Sie die Funktion setEventDescription(Name der Aarmvariablen, anzuzeigender Text) im Rückruf des Buttons (Go to slot...) zurückrufen, siehe Option "2" im Kapitel “Sprache aus dem Gerät auswählen”.

    Zum Beispiel: setEventDescription("Alarm_temp", "Temperaturalarm - Analogeingang 1");

    Folgen Sie diesem letzten Schritt für alle Sprachalarme.
