Descripción de la Crosstable

  • Sections

    La Crosstable está dividida en secciones, algunas disponibles para el usuario y otras bloqueadas y precompiladas con las variables del dispositivo.


    Sección

    Description

    Retentive

    Las variables de esta sección no pierden su valor después de apagar y encender el dispositivo Mect.

    Non Retentive

    Gran sección disponible para todas las variables de su proyecto que no necesitan ser “recordadas” después de apagar el dispositivo.

    Diagnostic

    Node Status

    Local I/O

    System

    Estas pueden ser modificadas parcialmente por el usuario y pueden utilizarse en sus proyectos; además de permitirle mover las E/S, proporcionan información importante sobre el funcionamiento interno del dispositivo Mect.

  • Campo

    Priority: Remote devices data exchange enable.

    Valor

    Descripción

    0

    No habilitado.

    1

    Habilitado con alta prioridad; preferiblemente para variables con protocolo “PLC”

    2

    Habilitado con media prioridad; preferiblemente para variables con protocolo “RTU”.

    3

    Habilitado con baja prioridad; preferiblemente para variables con protocolo “TCP”

    Update: Define el tipo de intercambio de datos que se activa con los dispositivos remotos.

    Valor

    Descripción

    H

    La variable se actualiza sólo si se muestra en la página actual.

    P

    La variable es siempre actualizada.

    S

    La variable siempre se actualiza y almacena lentamente con un tiempo de “Slow Log Period” (parámetro System Editor).

    F

    Como “S” pero de forma rápida y con un tiempo “Fast Log Period” (parámetro System Editor).

    V

    La variable se adquiere e historiza sólo cuando hay un cambioe

    X

    La variable es historizada cuando es llamada por la función logShot().

    Var.Name: Nombre de la variable. No puede tener más de 30 caracteres.


    Type: Se definen los siguientes tipos:


    Valor

    Descripción

    BIT

    Booleano.

    BYTE

    Byte.

    BYTE_BIT

    Selecciona un BIT dentro de un byte.

    WORD_BIT

    selecciona un BIT dentro de una palabra (16bit).

    DWORD_BIT

    Selecciona un BIT dentro de una palabra doble (32 bit).

    UINT

    Número entero sin signo de 16 bits.

    UINTBA

    Número entero sin signo de 16 bits con bytes invertidos.

    INT

    Número entero con signo de 16 bits.

    INTBA

    Número entero con signo de 16 bits con bytes invertidos.

    UDINT

    Número entero sin signo de 32 bits.

    UDINT....

    Número entero sin signo de 32 bits con bytes invertidos.

    DINT

    Número entero con signo de 32 bits.

    DINT....

    Número entero con signo de 32 bits con bytes invertidos.

    REAL

    Real.

    REAL...

    Real con bytes invertidos.

    Decimal: Para variables enteras o reales indica el número de decimales después de la coma con que se representa la variable. En caso de que la variable sea de tipo BYTE_BIT, WORD_BIT o DWORD_BIT indica la posición del BIT desde 1.


    Protocol: Indica el protocolo con el que se intercambia la variable:

    Valor

    Descripción

    PLC

    Variable de intercambio entre PLC y HMI.

    RTU

    Variable presente en un dispositivo remoto en Modbus RTU.

    TCP

    Variable presente en un dispositivo remoto a través de Modbus TCP.

    TCPRTU

    Variable presente en un dispositivo remoto en Modbus TCPRTU.

    CANOPEN

    Variable presente en un dispositivo CANopen remoto

    RTU_SRV

    Variable que responde a una solicitud de un master Modbus RTU.

    TCP_SRV

    Variable que responde a una solicitud de un master Modbus TCP.

    TCPRTU_SRV

    Variable que responde a una solicitud de un master Modbus TCPRTU.

    MECT

    Variable en el protocolo MECT.

    IP Address: Dirección IP para Modbus TCP y TCPRTU, por ejemplo: 192.168.0.42.


    Port: En el caso de variables NON PLC, indica qué puerto de comunicación utilizar: el puerto de comunicación depende del dispositivo utilizado.


    Node ID: Dirección del nodo del dispositivo remoto. En el caso de la dirección 0 la escritura es de tipo Broadcast.


    Register:irección del modbus del registro al que se quiere acceder.


    Block: Dirección del registro del primer bloque. Para los registros que forman parte de un bloque, la lectura debe estar fusionada.


    Comment: Se utiliza como campo de comentario para la variable, utilizar caracteres ASCII. No esta utilizado por el sistema.


    Behavior: Define si la variable es de tipo “lectura”, “lectura/escritura”, “allarme” o “evento”.


    Blk Size: Número de registros en el bloque, máximo 64 registros.
