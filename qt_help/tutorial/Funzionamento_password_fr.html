Fonctionnement des mots de passe

Il y a deux façons différentes de saisir les mots de passe:

  • Associer une variable dans la propriété PasswordVar d'un ATCMbutton ou insérer une constante directement pour protéger:

    • un changement de page.

    • l'action du bouton.

  • Dans toutes les pages créées, sauf celles de la librairie, il est possible d'associer des mots de passe système pour permettre l'accès.

    Le mécanisme du mot de passe est géré hiérarchiquement et il y a les niveaux suivants:

    • Admin

    • SuperUser

    • User

    • Operator

    Le niveau “Admin” est la priorité la plus élevée et “User” la priorité la plus faible, tandis que “Operator” indique qu'aucune connexion n'a été effectuée et qu'il n'y a pas de mot de passe actif (opération par défaut).

    Les mots de passe sont strictement numériques sans limitation particulière du nombre de chiffres.

    Dans l'application sur le panneau il y a deux touches impliquées dans la gestion des mots de passe:

    • Touche login sur la page login_pwd: permet de se connecter au système ou de définir le niveau d'accès actuel. Appuyez sur la touche pour afficher le clavier numérique dans lequel vous pouvez entrer le mot de passe. A partir de ce moment, les opérations sur l'interface graphique sont conditionnées au niveau d'accès de l'utilisateur authentifié. Le bouton de connexion se transforme en bouton de déconnexion. Pour vous déconnecter, appuyez simplement sur le nouveau bouton de déconnexion.

    • Touche change password présent dans la page login_pwd: permet de modifier le mot de passe de l'utilisateur actuellement connecté. En appuyant sur la touche, le pavé numérique apparaît pour entrer le nouveau mot de passe à associer au niveau utilisateur qui s'est connecté précédemment.

    Pour configurer et protéger n'importe quelle page du mot de passe, vous devez ouvrir la page correspondante .cpp.

    Pour protéger la page est suffisant au début du constructeur de page (page”N”::page”N”(QWidget *parent) ), juste sous la ligne:

    ui->setupUi(this);

    d'ajouter la ligne:

    protection_level = niveau minimum de connexion;

    Où “niveau minimum de connexion” correspond à:

    • pwd_admin_e
    • pwd_super_user_e
    • pwd_user_e
    • pwd_operator_e

    Par exemple, si je mets protection_level à pwd_user_e, je ne peux entrer dans la page que si je suis connecté avec le mot de passe “User”, “SuperUser” ou “Admin”. Sinon, une fenêtre contextuelle apparaîtra pour vous aviser que vous n'êtes pas autorisé à accéder à la page. N'oubliez pas d'enregistrer vos modifications sur la page.

    Si aucun niveau d'accès n'est spécifié dans le constructeur de page, la page a pwd_operator_e comme niveau d'accès par défaut, c'est-à-dire que la page est accessible par tous les niveaux d'utilisateur fournis et aussi par tout utilisateur qui ne s'est pas connecté au système.

    REMARQUE: Chaque utilisateur est autorisé à changer son propre mot de passe uniquement. En d'autres termes, l'utilisateur “User” ne peut changer que son mot de passe et l'utilisateur “Admin” ne peut pas modifier le mot de passe de “User”.

    La touche change password est désactivée jusqu'à ce qu'un utilisateur se connecte.

    La connexion n'expire pas tant que le panneau n'est pas redémarré ou si vous n'avez pas explicitement fermé la session en appuyant sur le bouton correspondant. Cependant, vous pouvez définir une durée d'inactivité maximale après laquelle la session de connexion expire. Directement à partir de “System Editor” il est possible de définir l'heure (Pwd Timeout [s]) et la page de destination en cas d'expiration de la session (Pwd Logout Page).
