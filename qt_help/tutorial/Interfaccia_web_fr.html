Fonctionnement de l'interface Web

Le panneau opérateur est équipé d'un serveur Web qui permet d'accéder avec le navigateur à une série de pages WEB à travers lesquelles vous pouvez vérifier certains paramètres du panneau ou effectuer des opérations de chargement et de téléchargement.

L'adresse IP définie dans le panneau peut être consultée dans l'IHM en accédant à la page info accessible depuis le menu principal de l'application de démonstration.

Pour utiliser l'interface Web, tapez l'adresse IP du panneau dans la barre URL du navigateur que vous utilisez.

Un mot de passe est requis pour accéder à l'interface WEB. Les mots de passe déterminent les paramètres d'accès aux fonctionnalités de l'interface WEB. Les valeurs par défaut sont définies comme suit:

  • User : 111111

  • SuperUser: 222222

  • Admin: 1357924680

Logs

La page Logs démarre le composant de l'interface web dédié à la gestion des historiques. Trois types d'opérations peuvent être effectuées sur les historiques :la mise à jour des définitions des historiques partiels et dans la page “Manage Logs” le le téléchargement et la suppression des historiques partiels et totaux.

Update LOG

La fonction de mise à jour des définitions d'historique partiel met à jour la liste des historiques partiels disponibles sur le panneau. Les définitions de l'historique à télécharger doivent être des fichiers au format CSV et le fichier doit commencer par le préfixe “store”:

date

temps

VARIABLE 1

…

VARIABLE N

L'interface télécharge un fichier CSV à la fois, donc pour télécharger plusieurs définitions, vous devez effectuer plusieurs mises à jour.

Downolad LOGS

L'interface Web vous permet de télécharger les logs de la façon suivante:

  • Total / Total partiel : en sélectionnant le log(journal) “Total” ou l'un des logs Partiel présent sans entrer aucune valeur dans les champs de la date, tous les logs présents sur le panneau sont téléchargés.

  • Jour Total / Jour Partiel : en sélectionnant le log “Total” ou l'un des logs Partiel présent et en entrant une valeur dans le champ “Initial Date” le log pour le jour spécifié sera téléchargé.

  • Totaux journaliers dans un intervalle de temps / Totaux journaliers partiels dans un intervalle de temps : en sélectionnant le le log “Total” ou l'un des logs Partiel présent et en entrantune valeur dans les champs “initial Date” et “FinalDate” les journaux journaliers stockés sur le panneau dans l'intervalle de temps spécifié par les dates indiquées seront téléchargés.

  • Total Unique en un intervalle de temps/Partiel Unique en un intervalle de temps: en sélectionnant le log “Total” ou l'un des partiels présents et en saisissant toutes les informations relatives à l'intervalle de temps “Initial Date” “Initial Time” “Final Date” et “Final Time” un seul fichier journal contenant toutes les données historisées dans l'intervalle spécifié est téléchargé.

Delete LOGS

L'interface Web vous permet de supprimer:

  • la définition des journaux partiels: l'annulation est obtenue en sélectionnant les logs partiels à supprimer sans introduire aucun paramètre de temps

  • tous les logs présents sur le système: l'annulation se fait en sélectionnant le log “Total” sans introduire aucun paramètre de temps

  • le log d'un jour donné: pour supprimer le log d'un jour donné, sélectionnez le log Total et entrez dans le champ “Initial Date” le jour pour lequel vous voulez supprimer les logs.

Les champs “Initial Time”, “Final Date” et “Final Time” doivent rester vides.

  • les log relatifs à un intervalle de temps : pour supprimer les journaux entre deux dates, sélectionnez log “Total” comme type de log et réglez les dates “Initial Date” et “Final Date”. Les champs “Initial Time” et “Final Time” doivent être laissés vides.

REMARQUE:pour les logs partiels, il n'est pas possible d'effectuer des annulations pour un jour ou un intervalle de temps car les journaux partiels ne sont générés à partir des logs totaux que lorsque l'utilisateur demande le téléchargement.

Trend

La page Trend démarre la partie interface dédiée à la gestion des tendances. Il y a quatre types d'opérations qui peuvent être effectuées sur les trends: la mise à jour, le téléchargement de captures d'écran et dans la page “Manage Trend” le téléchargement et la suppression.

Update

La fonction de mise à jour des familles de trends met à jour la liste des familles de trends disponibles sur le panneau. Les familles de trends à télécharger doivent être compressées dans une archive au format zip. L'archive est composée d'une liste de fichiers, dont chacun décrit un type de trend

Manage trend

Après avoir téléchargé les trendes selon la structure indiquée, la page Manage Trend vous permet de naviguer dans les dossiers contenant les familles de trends elles-mêmes. Il est possible de télécharger ou de supprimer une famille de trends ou une série de familles à sélections multiples.

Alarms

La page Alarmes démarre le composant d'interface web dédié à la gestion des alarmes.
Deux types d'opérations peuvent être effectuées sur les alarmes : le téléchargement et la suppression des logs sur le panneau.

Recipes

La page Recettes démarre la section de l'interface dédiée à la gestion des recettes.

Trois types d'opérations peuvent être effectuées sur les recettes: la mise à jour et dans la page “Manage Recipes” le téléchargement et la suppression.

Update

La fonction de mise à jour des recettes met à jour la liste des recettes disponibles sur le panneau. Les recettes à télécharger doivent être compressées dans une archive au format zip. L'archive est organisée en dossiers dont chacun correspond à un type de recette et, à l'intérieur de chaque dossier, il y a les différentes implémentations d'un type de recette.

Par exemple, si nous supposons que nous avons deux familles de recettes, l'une pour le refroidissement et l'autre pour le chauffage dans les archives contenant les recettes, nous aurons l'organisation suivante:

├── cool

│ ├── cool_10

│ ├── cool_20

│ └── cool_30



└── hot

├── hot_100

├── hot_150

├── hot_200

└── hot_50

Manage recipes

Après avoir téléchargé les recettes selon la structure indiquée, la page Manage Recipes les recettes vous permet de naviguer dans les dossiers contenant les recettes elles-mêmes. Il est possible de télécharger ou de supprimer des recettes sur l'ensemble de la famille de recettes en sélectionnant la famille, soit sur une seule recette appartenant à une famille, soit sur plusieurs sélections de recettes appartenant à la même famille.

LAN Config

Dans cette page, vous pouvez définir les propriétés réseau du panneau. Le panneau ne supporte qu'une configuration réseau statique pour laquelle vous pouvez définir l'adresse IP, le masque réseau (NM), la passerelle par défaut (GW) et les serveurs DNS (DNS1 et DNS2).

Date and Time

La page “Date and Time” détecte automatiquement la date et l'heure du PC sur lequel tourne le navigateur qui se connecte au panneau et vous permet de régler les valeurs détectées sur le panneau lui-même.

Display Settings

La page Display Settings vous permet de modifier les fonctions suivantes ::

  • Luminosité de l'écran : il est possible de régler une valeur de luminosité entre 0 et 100 correspondant respectivement à la luminosité minimale et maximale autorisée.

  • Economiseur d'écran: Vous pouvez régler le temps d'arrêt de l'affichage après lequel l'économiseur d'écran du système est activé. La valeur réglée doit être exprimée en secondes. Une valeur de 0 désactive 0 l'économiseur d'écran.

  • Touch Calibration: &Vous pouvez lancer la procédure d'étalonnage de l'écran tactile associé à l'affichage. La procédure d'étalonnage interrompt l'exécution du logiciel IHM et API normalement actif sur le panneau et exige que l'utilisateur touche les points surlignés sur l'écran. A la fin du calibrage, l'IHM et le logiciel du PLC redémarrent automatiquement.

Manage Configuration

Permet à l'utilisateur de mettre à jour l'écran d'accueil (splash.png) qui peut être affiché au début de l'application IHM.

Manage Password

Permet à l'utilisateur qui s'est connecté à l'interface avec le mot de passe “User” ou “Super User” de changer uniquement son mot de passe. Le mot de passe associé au niveau “Admin” ne peut pas être modifié. REMARQUE: Ces mots de passe sont différents des mots de passe utilisés dans l'application IHM.

Reset Passwords

La fonction de réinitialisation des mots de passe rétablit les mots de passe de l'interface WEB à la valeur par défaut indiquée ci-dessus. Il n'est accessible qu'à l'utilisateur “Admin”.
