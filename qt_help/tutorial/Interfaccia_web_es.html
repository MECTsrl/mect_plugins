Funcionamiento de la interfaz web

El Panel de Operador está equipado con un servidor web que permite el acceso con el navegador a una serie de páginas WEB a través de las cuales se puede comprobar la configuración del panel o realizar operaciones de carga y descarga.

La dirección IP establecida en el panel se puede ver en la HMI accediendo a la página info a la que se accede desde el menú principal de la aplicación de demostración.

Para utilizar la interfaz web, escriba la dirección IP del panel en la barra de direcciones del navegador que está utilizando.

Se requiere una contraseña para acceder a la interfaz WEB. Las contraseñas determinan los permisos de acceso a las funcionalidades de la interfaz web. Los valores por defecto se fijan de la siguiente manera:

  • User : 111111

  • SuperUser: 222222

  • Admin: 1357924680

  • LAN Config

    En esta página puede configurar las propiedades de red del panel. El panel sólo admite una configuración de red estática para la que se puede definir la dirección IP, la máscara de red (NM), la pasarela por defecto (GW) y los servidores dns (DNS1 y DNS2).

  • Date and Time

    La página “Date and Time” detecta automáticamente la fecha y la hora del PC en el que se ejecuta el navegador que se conecta al panel y permite configurar los valores detectados en el propio panel.

  • Display Settings

    La página Display Settings le permite cambiar las siguientes características:

    • Brillo de la pantalla: es posible establecer un valor de brillo entre 0 y 100 correspondiente al brillo mínimo y máximo permitido respectivamente.

    • Salvapantallas: Puede establecer el tiempo de inactividad de la pantalla después del cual se activa el salvapantallas del sistema. El valor ajustado debe expresarse en segundos. Un valor de 0 desactiva el salvapantallas.

    • Touch Calibration: puede iniciar el procedimiento para calibrar la pantalla táctil asociada a la pantalla. El procedimiento de calibración interrumpe la ejecución del software HMI y PLC normalmente activo en el panel, y requiere que el usuario toque los puntos resaltados en la pantalla. Al final de la calibración, el software de la HMI y del PLC se reiniciarán automáticamente

  • Manage Configuration

    Permite al usuario actualizar la pantalla de bienvenida (splash.png) que se puede mostrar al inicio de la aplicación HMI.

  • Alarms

    La página Alarmas inicia el componente de interfaz web dedicado a la gestión de alarmas.
    Hay dos tipos de operaciones que se pueden realizar en las alarmas: descargar y borrar los logs en el panel.

  • Logs

    La página Logs inicia el componente de la interfaz web dedicado a la gestión de historiales. Existen dos tipos de operaciones que se pueden realizar sobre los historiales: la descarga y eliminación de los historiales.

  • Trend

    La página Trend inicia la parte de la interfaz dedicada a la gestión de trends. Existen tres tipos de operaciones que se pueden realizar sobre los trends: la actualización,la descarga y el borrado.

    Upload:

    La función de actualización de la familia de trends actualiza la lista de familias de trends disponibles en el panel. Las familias de trends que se van a cargar deben comprimirse en un archivo en formato zip. El archivo se compone de una lista de ficheros, cada uno de los cuales describe un tipo de trend.

  • Recipes

    La página Recetas inicia la sección de la interfaz dedicada a la gestión de recetas.

    Hay tres tipos de operaciones que se pueden realizar en las recetas: la actualización, la descarga y el borrado.

    Upload:

    La función de actualización de recetas actualiza la lista de recetas disponibles en el panel. Las recetas que se van a cargar deben estar comprimidas en un archivo en formato zip. El archivo comprimido está organizado en carpetas, cada una de las cuales corresponde a un tipo de receta y, dentro de cada carpeta,se encuentran las distintas implementaciones de un tipo de receta.

    Por ejemplo, si suponemos que tenemos dos familias de recetas, una para enfriar y otra para calentar dentro del archivo que contiene las recetas, tendremos la siguiente organización:

    ├── cool

    │ ├── cool_10

    │ ├── cool_20

    │ └── cool_30



    └── hot

    ├── hot_100

    ├── hot_150

    ├── hot_200

    └── hot_50

  • Screenshot

    La página Screenshot inicia el componente de interfaz web dedicado a la gestión de capturas de pantalla.
    Hay dos tipos de operaciones que se pueden realizar en las Screenshot: descargar y borrar.

  • Manage Password

    Permite al usuario que ha iniciado sesión en la interfaz con la contraseña “User” o “Super User” cambiar sólo su contraseña. La contraseña asociada al nivel “Admin” no se puede modificar. NOTA: Estas contraseñas son diferentes de las utilizadas en la aplicación HMI.
