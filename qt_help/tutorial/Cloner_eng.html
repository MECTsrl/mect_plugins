Cloner

The “Cloner” is used to make a copy of an application. It could be a backup copy and/or it could be used to update other devices (of the same model).

BACKUP procedure:

  • Copy img_cloner_3.x.x.ext2 and sysupdate_cloner_3.x.x.sh files in a USB pen drive (FAT32 with one and only one ms-dos MBR partition).
    Warning: do NOT use “Edge” browser because it changes the file extension.
    Warning: there must not be any other update files: sysupdate_....

  • Turn off the device.

  • Plug in the USB pen drive (if the device has a micro-USB port, use a micro-USB (A type) → USB adaptor).

  • Turn on the device.

  • Press the “BACKUP” button.

  • Write the name you want and press

  • BACKUP process is ended when “Operation ‘Local file system’ completed.” appears.

  • Turn off the device.

  • Take off the USB pen drive.

The BACKUP copy will be in the “cloner” folder in the pen drive.


INSTALL procedure:

  • Copy in a USB pen drive (FAT32 with one and only one ms-dos MBR partition):

    • the img_cloner_3.x.x.ext2 and sysupdate_cloner_3.x.x.sh files.
      Warning: do NOT use “Edge” browser because it changes the file extension.

    • the “cloner” folder.
      Warning: inside the “cloner” folder there must be a folder with the name of the “backup” that was done in the BACKUP procedure. The file “localfs.tar” must be located in this folder.

                 

  • There must be no other update files: sysupdate_…

  • Turn off the device.

  • Plug in the USB pen drive (if the device has a micro-USB port, use a micro-USB (A type) → USB adaptor).

  • Turn on the device.

  • Select the version you want to install from the drop-down menu.

  • Press the “INSTALL” button.

  • Install process is ended when “Operation ‘Local file system’ completed.” appears.

  • Turn off the device.

  • Take off the USB pen drive.
