Realisierung einer Automatisierung

Die Realisierung einer grafikbezogenen Automatisierung kann auf zwei Arten (auch gleichzeitig) erfolgen:

  • In QT-Umgebung, die einen Code C++ realisiert.

  • In der ATCM Control Umgebung durch die 5 Sprachen, die in SPSen üblich sind

In beiden Fällen erfolgt der Austausch zwischen Automatisierung und Grafik über den Crosstable.

Obwohl es in Qt nicht notwendig ist, C++-Code zu schreiben,  ist dies dennoch möglich, z.B. wenn Sie einige SPS-Variablen aus der HMI-Anwendung direkt manipulieren möchten.

HINWEIS: Dies geschieht in der Regel in der SPS-Anwendung selbst.

Die Qt-Automatisierungen können für das gesamte Projekt oder für jede einzelne Seite durchgeführt werden.

Um eine Variable aus C++ zu schreiben, rufen Sie die Funktion doWrite_ VariableName(Wert) auf, wobei VariableName der Name der in Crosstable vorhandenen Variable und Wert der zu schreibende Wert ist.

Zum Beispiel, um das erste digitale E/A wie ausgang zu konfigurieren schreiben Sie:

doWrite_PLC_DigDir_1(1);

Um dem ersten Analogausgang das Produkt aus den Werten der ersten beiden Analogeingänge zuzuordnen, schreiben Sie:

doWrite_PLC_AnOut_1(PLC_AnIn_1 * PLC_AnIn_2);


Es ist möglich, die folgende Struktur zu verwenden, um mehrere Variablen gleichzeitig zu schreiben:

beginWrite();

addWrite_Variable_Name_1(Value);

addWrite_Variable_Name_2(Value);

addWrite_Variable_Name_N(Value);

endWrite();

HINWEIS: die Schreibvorgänge mit den Funktionen doWrite und addWrite DÜRFEN NICHT STÄNDIG aufgerufen werden, sondern nur auf die Differenz des Wertes.


GESAMTPROJEKT

Sie können Ihre eigene Anwendung nur mit C++-Code schreiben; dazu müssen Sie die Seite automation.cpp in Edit → Sources verwenden

Diese Seite ist in zwei Teile gegliedert: setup() und loop().

Die setup()-Funktion wird nur am Anfang der Anwendung aufgerufen, während die loop()-Funktion alle 100 ms unabhängig von der angezeigten Seite aufgerufen wird.


EINSEITIGE SEITE

Um eine Automatisierung in einer QT-Umgebung zu realisieren, die nur auf einer Seite variiert, folgen Sie den drei unten aufgeführten Schritten:

  1. Legen Sie die Aktion fest, die die Automatisierung auslöst: eine Zeit, ein erreichter Schwellenwert, der Druck einer Taste, etc...

  2. Binden Sie die gewählte Aktion mit der Funktion, die die Automatisierungsbefehle beinhaltet, an den vorherigen Punkt.

  3. Füllen Sie die Funktion aus, die die Automatisierung implementiert.

Nachfolgend finden Sie ein Beispiel, in dem Sie den Druck einer Taste mit der Aktion der Umkehrung einer Reihe von 2 Bits verknüpfen möchten.

Zuerst müssen Sie die Taste mit der Automatisierungsfunktion (in Qt bekannt als Slot) und der Druckaktion (in Qt bekannt als Signal) verbinden.

Klicken Sie dazu mit der rechten Maustaste auf die Schaltfläche und wählen Sie den Eintrag “go to slot” " und wählen Sie aus der angezeigten Liste das Signal “clicked”. Es erscheint ein Fenster mit einer leeren Automatisierungsfunktion.



Wenn das folgende Fenster erscheint:

Sie müssen ein “Build All” durchführen und dann das Projekt schließen, indem Sie File → “Close All Projects and Editors” wählen. Wenn das Projekt wieder geöffnet wird, sollte das Fehlerfenster nicht mehr angezeigt werden.

/* this funcion will flip the actual value of DigOut01, DigOut02, DigOut03 and DigOut04 */
void page01::on_atcmButton_5_clicked()
{
    /***********
    * DigOut01 *
    ************/
    /* flip the value */
    doWrite_DigOut01(!DigOut01);
        /***********
    * DigOut02 *
    ************/
    /* flip the value */
    doWrite_DigOut02(!DigOut02);
}

Darüber hinaus kann der Quellcode jeder Seite mit verschiedenen Methoden zugeordnet werden:

  1. in der Methode reload(), die nur aufgerufen wird, wenn die Seite auf der Anzeige erscheint;

  2. in der Methode updateData(), die alle 500 ms aufgerufen wird, wenn die Seite sichtbar ist;
