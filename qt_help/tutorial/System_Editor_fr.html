Éditeur système

Avec l'éditeur de système, il est possible de modifier certaines configurations pour l'interface graphique et les paramètres de communication entre le panneau et les nœuds distants.

Pour ouvrir l'“éditeur de système”, sélectionnez l'icône MectSuite:


Voici l'“éditeur de système”:

Dans la partie supérieure de l'onglet “System” se trouve le bouton pour enregistrer les réglages. Ci-dessous se trouvent quelques onglets : System, SERIAL PORT 0 etc….qui peuvent être édités ou non en fonction des ports disponibles sur le produit.

Voici les caractéristiques de “l'éditeur de système”:

  • System: : toutes les caractéristiques générales du système sont définies:

    • Pwd Logout Page: Page affichée à l'expiration de la session. Si la zone est vide, l'écran actuel reste affiché.

    • Pwd Timeout [s]: : Indique après combien de secondes d'inactivité la session expire. Avec la valeur “0”, la session n'expire jamais.

    • Screen Saver [s]: : Indique après combien de secondes le mode “Economiseur d'écran” commence à fonctionner . Avec la valeur “0”,le réglage est désactivé.

    • Slow Log Period [s]: Indique combien de secondes vous voulez stocker les variables de type “S” (définies dans le champ “Update” de “l'éditeur de Crosstable”).

    • Fast Log Period [s]: Indique combien de secondes vous voulez stocker les variables de type “F” (définies dans le champ “Update” de “l'éditeur de Crosstable”). Le nombre doit être inférieur au champ “Slow Log Period”.

    • Max Log Space [Mb]: Quantité d'espace dédiée à la sauvegarde des variables historisées.

    • Trend Window [s]: Contient la valeur de la fenêtre de temps d'affichage des trends. Le nombre doit être au moins trois fois supérieur au champ “Fast Log Period”.

    • Retries: indique le nombre d'erreurs permises à un appareil avant de réduire le taux d'interrogation.

    • Blacklist: Taux d'interrogation réduit en cas de dépassement du nombre défini dans le champ “Retries”.

    • Start Page: Nom de la page de l'écran que vous voulez afficher en premier. Dans Qt Creator doit être présente une page avec ce nom. Seules les pages présentes à l'ouverture du projet sont affichées.

    • Home Page: Nom de la page que vous souhaitez associer à la “Home Page”. Dans Qt Creator doit être présente une page avec ce nom. Seules les pages présentes à l'ouverture du projet sont affichées.

    • Buzzer Touch: : Indique si un signal sonore doit être associé à chaque événement de pression sur l'écran tactile.

    • Buzzer Alarm: Indique si l'apparition d'une condition d'alarme doit être associée à un signal sonore.

    • Read Period 1 [ms]: indique la fréquence de lecture des variables ayant la valeur “1” dans le champ “Priority” de la Crosstable.

    • Read Period 2 [ms]: indique la fréquence de lecture des variables ayant la valeur “2” dans le champ “Priority” de la Crosstable.

    • Read Period 3 [ms]: indique la fréquence de lecture des variables ayant la valeur“3” dans le champ “Priority” de la Crosstable.

      Le champ “Priority” n'est pas pris en compte pour les variables avec protocole PLC.

  • SERIAL PORT 0 / 1 / 2 / 3:Réglage de tous les paramètres de communication de l'appareil série.

    Dans le champ “Max Block Size” vous pouvez définir le nombre maximum de variables dans un bloc.

    ATTENTION:la réorganisation des blocs a lieu après l'enregistrement de la Crosstable. Les paramètres séries internes ne doivent pas être modifiés, sinon l'appareil ne fonctionnera pas correctement.

  • TCP IP PORT: vous définissez les paramètres de la communication TCP IP avec un appareil distant:

    • Timeout [ms]: temps d'attente maximum pour que le Master reçoive une réponse du Slave.

    • Silence [ms]: fréquence des requêtes consécutives d'un Master à un Slave.

    • Max Block Size [1…64]: nombre maximum de variables dans un bloc. ATTENTION: la réorganisation des blocs a lieu après l'enregistrement de la Crosstable.

  • CANOPEN 0 / 1: Réglages de la vitesse de transmission du protocole CANopen.
