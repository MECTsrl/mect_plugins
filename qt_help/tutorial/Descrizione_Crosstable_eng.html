Description of Crosstable fields

  • Sections

    The Crosstable is divided into several sections, some modifiable from the user and others blocked and precompiled with the variables of the device.


    Section

    Description

    Retentive

    The variables in this section do not lose their value after turning the Mect device off and on again.

    Non Retentive

    A large section available for all the variables of your project that do not need to be “remembered” after the device has been switched off.

    Diagnostic

    Local I/O

    System

    These are partially editable by the user and can be used in your projects, as well as allowing you to move the I / O, they provide important information on the internal operation of the Mect device.

  • Fields

    Priority: Remote devices data exchange enable.

    Value

    Description

    0

    Disabled.

    1

    High priority enable, preferably for variables with “PLC” protocol

    2

    Medium priority enable, preferably for variables with “RTU” protocol

    3

    Low priority enable, preferably for variables with “TCP” protocol.

    Update: It defines the type of data exchange enabled with the remote devices.

    Value

    Description

    H

    The variable is updated only if it appears in the current page.

    P

    The variable is always updated

    S

    The variable is always updated and logged slowly with time “Slow Log Period” (System Editor parameter).

    F

    Like “S” but quickly with time “Fast Log Period” (System Editor parameter).

    V

    Variable is stored only on change

    X

    Variabile is stored when logShot() function is called.

    Var.Name: Name of the variable (30 char max and different by “Text”).


    Type: The following types are defined:


    Value

    Description

    BIT

    Boolean.

    BYTE

    Byte.

    BYTE_BIT

    Select a BIT of a byte.

    WORD_BIT

    select a BIT of a word (16 bit).

    DWORD_BIT

    Select a BIT of a double word (32 bit).

    UINT

    Unsigned integer 16 bit.

    UINTBA

    Unsigned integer 16 bit swapped byte.

    INT

    Integer 16 bit.

    INTBA

    Integer 16 bit swapped byte.

    UDINT

    Unsigned integer 32 bit.

    UDINT....

    Unsigned integer 32 bit swapped byte.

    DINT

    Integer 32 bit.

    DINT....

    Integer 32 bit swapped byte.

    REAL

    Real.

    REAL...

    Real swapped byte.

    Decimal: For integer or real variables is the number of decimal values of the variables. For variables BYTE_BIT, WORD_BIT or DWORD_BIT is the bit position starting from 1.


    Protocol: Type of protocol to be used with the remote device:

    Value

    Description

    PLC

    Variable for exchange between PLC and HMI.

    RTU

    Variable on a remote device on Modbus RTU

    TCP

    Variable on a remote device on Modbus TCP

    TCPRTU

    Variable on a remote device on Modbus TCPRTU

    CANOPEN

    Variable on a remote device on CANopen

    RTU_SRV

    Variable for a master Modbus RTU

    TCP_SRV

    Variable for a master Modbus TCP

    TCPRTU_SRV

    Variable for a master Modbus TCPRTU.

    MECT

    Variable on MECT protocol.

    IP Address: IP address for modbus TCP and TCPRTU, for example: 192.168.0.42 .


    Port: For NON PLC variables is the communication port to use: it depends on used device.


    Node ID: Address of the remote device node. In case of 0 address the writing is Broadcast.


    Register: Modbus register address.


    Block: Address of the first register of the block. For registers belonging to the same block only one reading request is issued.


    Comment: Variable comment field. Not used by system.


    Behavior: Defines if variable is “Read”, “Read/Write”, “Alarm” or “Event”.


    Blk Size: Number of registers available in the block. Max 64 registers.
