Password function

There are two different ways to enter the password:

  • Insert in the PasswordVar property of an ATCMbutton a variable, or directly enter a constant to protect:

    • a page change

    • the action of the button.

  • In each created page, except for library ones, it is possible to create a system password to access to.

    Password are hierarchical and the levels are:

    • Admin

    • SuperUser

    • User

    • Operator

    “Admin” level has the highest priority and “User” the less one, while “Operator” means that there is no login performed then no active password (default status).

    Password are strictly numbers with no restriction on digit numbers.

    In the operator panel application there are 2 keys for password managing:

    • login key in the login_pwd page: allows system login or to setup current level access. Clicking on key the numeric pad appears to enter password. From now on the operations on the graphic interface are conditioned by user level access. Login key becomes logout. To perform logout, press logout key.

    • change password key in login_pwdpage: allows password change of the current logged user. Clicking on key the numeric pad appears to enter the new password for the logged user.

    To setup and protect a page with password it is necessary to open the .cpp page.

    To protect the page, at the beginning of the page constructor (page”N”::page”N”(QWidget *parent) ), immediately under the row:

    ui->setupUi(this);

    add the row:

    protection_level = minimum login level;

    where “minimum login level” is:

    • pwd_admin_e
    • pwd_super_user_e
    • pwd_user_e
    • pwd_operator_e

    For example, if protection_level is set to pwd_user_e, I can enter the page only if logged with “User”, “SuperUser” or “Admin” password. Otherwise a popup window shows a permission denied to enter the page. Remember to save the page after modifying it.

    If in the page constructor there is no access level specified, default level is pwd_operator_e so all users can access the page even if not logged.

    WARNING: each user can modify only its own password: “User” can modify its password and “Admin” can’t modify “User” password.

    Change password key is disabled until a user performs a login.

    Login doesn’t expires until panel restart or until logout. However, you can set a maximum idle time after which the login session expires. Directly from the “System Editor” it is possible to set the time (Pwd Timeout [s]) and the destination page in case the session expires (Pwd Logout Page).
