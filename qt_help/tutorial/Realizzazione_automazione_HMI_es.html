Realización de una automatización

La realización de una automatización relacionada con los gráficos se puede hacer de dos maneras (incluso simultáneamente):

  • En entorno QT realizando algún código C++

  • En el entorno ATCM Control a través de los 5 lenguajes comúnmente utilizados en los PLCs

En ambos casos, el intercambio entre la automatización y los gráficos se realiza a través de la Crosstable.

Aunque en Qt no es necesario escribir código C++,  es posible hacerlo, por ejemplo si se desea manipular directamente algunas variables del PLC desde la aplicación HMI.

NOTA: Esto se hace normalmente en la propia aplicación de PLC.

Las automatizaciones Qt se pueden hacer para todo el proyecto o para cada una de las páginas.

Para escribir una variable desde C++ llamar a la función doWrite_NombreVariable(valor), donde NombreVariable es el nombre de la variable presente en Crosstable y valor es el valor a escribir.

Por ejemplo, para configurar en salida la primera E/S digital escribir:

doWrite_PLC_DigDir_1(1);

Para asignar a la primera salida analógica el producto de los valores de las dos primeras entradas analógicas se debe escribir:

doWrite_PLC_AnOut_1(PLC_AnIn_1 * PLC_AnIn_2);


Es posible usar la siguiente estructura para escribir múltiples variables al mismo tiempo:

beginWrite();

addWrite_Variable_Name_1(Value);

addWrite_Variable_Name_2(Value);

addWrite_Variable_Name_N(Value);

endWrite();

NOTA: las escrituras con las funciones doWrite y addWrite NO DEBEN ser llamadas CONTINUAMENTE sino solo en la diferencia del valor.


PROYECTO COMPLETO

Puede escribir su propia aplicación usando sólo código C++; para ello necesita usar la página automation.cpp en Edit → Sources

Esta página está dividida en dos partes: setup() y loop()

La función setup() se llama sólo al inicio de la aplicación, mientras que la función loop() se llama cada 100ms, independientemente de la página mostrada.


PÁGINA ÚNICA

Para realizar una automatización en un entorno QT que sólo varía, por ejemplo, en una página, siga los tres pasos que se indican a continuación:

  1. Establecer la acción que activa la automatización: un tiempo, un umbral alcanzado, la presión de un botón, etc...

  2. Atar la acción elegida al punto anterior con la función que incorporará los comandos de automatización.

  3. Rellene la función que implementará la automatización.

A continuación se muestra un ejemplo en el que se desea vincular la presión de un botón a la acción de invertir una serie de 2 bits.

En primer lugar hay que atar el botón a la función de automatización (en Qt conocido como ranura) a la acción de la presión (en Qt conocido como señal).

Para ello, haga clic con el botón derecho del ratón sobre el botón y seleccione la opción “go to slot” " y seleccione de la lista mostrada la señal “clicked”. Aparecerá una ventana con una función de automatización vacía.



Si aparece la siguiente ventana:

Necesita hacer un “Build All” y luego cerrar el proyecto seleccionando File → “Close All Projects and Editors”. Cuando se vuelve a abrir el proyecto, la ventana de error ya no debería aparecer.

/* this funcion will flip the actual value of DigOut01, DigOut02, DigOut03 and DigOut04 */
void page01::on_atcmButton_5_clicked()
{
    /***********
    * DigOut01 *
    ************/
    /* flip the value */
    doWrite_DigOut01(!DigOut01);
        /***********
    * DigOut02 *
    ************/
    /* flip the value */
    doWrite_DigOut02(!DigOut02);
}

Además, el código fuente puede ser asociado con cada página usando diferentes métodos:

  1. en el método reload(), que sólo se llama cuando la página aparece en la pantalla;

  2. en el método updateData(), que se llama cada 500 ms cuando la página está visible;
