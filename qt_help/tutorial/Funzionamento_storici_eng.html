History functioning

Panel stores all data of the Crosstable values with “Update” field at:

  • S: variable is always updated and stored slowly with time “Slow Log Period” (System Editor parameter);

  • F: like “S” but fast with time “Fast Log Period” (System Editor parameter);

  • V: variable is sampled and stored only on change value;

  • X: variable is stored only when function logShot() is called.

History is daily and unique for all variables enabled for this function. It is possible to define partial history to display on panel.

To show history page select MENU → DATA → STORE, or use an ATCMbutton in the change page mode and insert in the Property “pageName” word “file name” without extension. Where “file name” is a .csv file beginning with “store”.

Upload the new file on the device via web interface, see “Web interface functioning”. Optionally it is possible to define a subset of the history variables writing a configuration file (store1.csv) in this way:

date

time

TPLC_IN_3

TPLC_IN_AN_2

Key allows time filtering of history and key, active when USB pen inserted, allows USB pen storage.

NOTE: log function can be started, stopped and forced using the functions logStart(), logStop() e logShot().

logReset() function removes all the logs on the device. These functions are inserted in the code al explained in chapter “Setting up an automation”.
