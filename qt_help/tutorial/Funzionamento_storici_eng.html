History functioning

Panel stores all data of the Crosstable values with “Update” field at:

  • S: variable is always updated and stored slowly with time “Slow Log Period” (System Editor → System parameter);

  • F: like “S” but fast with time “Fast Log Period” (System Editor → System parameter);

  • V: variable is sampled and stored only on change value;

  • X: variable is stored only when function logShot() is called.

Log function can be started, stopped and forced using the functions logStart();, logStop(); e logShot();. logReset(); function removes all the logs on the device. These functions are inserted in the code al explained in chapter “Setting up an automation”.

History is daily and unique for all variables enabled for this function. A new file is generated at logStart(); (a new file is automatically created at 00.00).

To show history page use an ATCMbutton and insert in the Property “pageName” the name of the library page “store”.

Key allows time filtering of history and key, active when USB pen inserted, allows USB pen storage.


It is possible to define partial history to display on panel.
