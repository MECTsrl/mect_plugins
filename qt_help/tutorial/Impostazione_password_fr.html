Définition des mots de passe

Il existe 2 façons différentes d'afficher/définir les mots de passe:

  • Utilisez la page dédiée de la bibliothèque “set_password”.

  • Utilisez la page générique de la bibliothèque “menu”.

Utilisation de la page “set_password”:

Pour afficher la page de configuration du mot de passe, utilisez un ATCMbutton et entrez le nom de la page de la bibliothèque “set_password” dans la propriété “pageName”.

Il est conseillé de protéger le changement de page du ATCMbutton par un mot de passe différent, statique ou dynamique, en l'insérant dans le champ “PasswordVar”.

Dans la page affichée, il est possible de définir les mots de passe des 3 niveaux: “Admin”, “SuperUser” et “User”.


Utilisation de la page “menu”:

Tous les systèmes ont un mot de passe par défaut de “0” pour les trois niveaux.

Comme les mots de passe par défaut sont identiques pour les trois niveaux, la première opération de login effectuée par l'application est gérée de manière hiérarchique à partir du niveau de priorité le plus élevé. Par conséquent, le premier login effectué dans l'application sera attribué à “Admin” et toute pression sur la touche ‘Password’ changera le mot de passe de “Admin”.

Le login suivant avec le mot de passe “0” sera attribué à l'utilisateur “SuperUser” dont le mot de passe peut être modifié si nécessaire. Le login suivant avec le mot de passe “0” sera attribué à l'utilisateur “User” dont le mot de passe peut être modifié à volonté.

Le dernier login avec le mot de passe “0” sera attribué à “Operator”.
