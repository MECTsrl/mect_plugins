Crosstable Editor functionality

  • Goto Section: allows navigation inside the Crosstable.

    • Retentive [1 - 192]: variables that mantain their values on power off;

    • Not Retentive [193 - 4999]: variables that loose their values on power off;

    • Diagnostic [5000 - 5299]: diagnostic variabiles. Used if a different protocol from “PLC” is used to communicate. See more in chapter “Diagnostic variabiles”;

    • Local I/O [5300 - 5389]: device I/O variables. Can be renamed and modified (in part) to allow more flexibility during writing application. See more in chapter “Device variabiles”;

    • System [5390 - 5472]: system variables. See more in chapter “Device variabiles”.

  • : extends or restrict the empty rows of the Crosstable.

  • : looks for a variable inside a Crosstable.

  • : load the “Crosstable.csv” file of another project. Imported file overwrite previous one when saved.

  • : undo last action.

  • : save actual setup. Red if a change is pending, green otherwise.

  • : checks errors in table.

  • : runs PLC program. If no error occurr in the Crosstable it is possible to open the PLC software. See more in chapter “Open the PLC software”.

It is possible do some operations by right-clicking the mouse:

  • Paste MPNC005 / MPNC006 Modules: paste all the variables associated with the MPNC005 / MPNC006 remote models and (MPNC020 / MPNC030 / MPNC035) connected modules.

  • Paste TPLC050 Modules: paste all the variables associated with the (MPNC020 / MPNC030 / MPNC035) connected modules.


ALARM / EVENT variable:

If a variable must be an alarm/event one it is necessary to define it in the “Behavior” field but it is possible only if:

  • Priority” field is not “0”;

  • Update” field is not “H”;

  • Type” field is equal to “BIT”;

With this setup it is possible to choose the alarm/event condition.

Variables in the drop menu are all present in the Crosstable and system variables but not those with “Update” fields equal to “H” and those with “Priority” field equal to “0”.
