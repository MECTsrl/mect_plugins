Passwörter Funktionsweise

Es gibt zwei verschiedene Möglichkeiten, Passwörter einzugeben:

  • Verknüpfen Sie eine Variable in der PasswordVar-Eigenschaft einer ATCMbutton oder geben Sie eine Konstante direkt zum Schutz ein:

    • ein Seitenwechsel.

    • die Aktion der Taste.

  • Auf jeder erstellten Seite, mit Ausnahme derjenigen der Bibliothek, ist es möglich, Systempasswörter zuzuordnen, um den Zugriff zu ermöglichen.

    Der Passwortmechanismus wird hierarchisch verwaltet und es gibt die folgenden Ebenen:

    • Admin

    • SuperUser

    • User

    • Operator

    Die Ebene “Admin” ist die höchste Priorität und “User” die niedrigste Priorität, während “Operator” anzeigt, dass keine Anmeldung erfolgt ist und kein aktives Passwort vorhanden ist (Standardbetrieb).

    Passwörter sind streng numerisch, ohne besondere Begrenzung der Anzahl der Ziffern.

    In der Anwendung auf dem Bedienfeld sind zwei Tasten für die Verwaltung von Passwörtern vorgesehen:

    • Login-Taste auf der login_pwd: ermöglicht es Ihnen, sich am System anzumelden oder die aktuelle Zugriffsstufe einzustellen. Durch Drücken der Taste wird der Ziffernblock zur Eingabe des Passworts angezeigt. Von diesem Zeitpunkt an werden die Operationen auf der grafischen Oberfläche auf die Zugriffsebene des authentifizierten Benutzers konditioniert. Die Login-Tatse verwandelt sich in eine Logout-Tatse. Um sich abzumelden, drücken Sie einfach die neue Logout-Taste.

    • change password-Tatse auf der login_pwd: Er ermöglicht es, das Passwort des aktuell angemeldeten Benutzers. Durch Drücken der Taste erscheint der Ziffernblock, in dem Sie das neue Passwort eingeben können, das der zuvor angemeldeten Benutzerebene zugeordnet werden soll.

    Um eine beliebige Seite einzurichten und vor dem Passwort zu schützen, müssen Sie die entsprechende page .cpp öffnen.

    Um die Seite zu schützen, reicht es am Anfang des Page Builders (page”N”::page”N”(QWidget *parent) ), direkt unter der Zeile:

    ui->setupUi(this);

    fügt die Zeile hinzu:

    protection_level = minimaler Login-Level;

    wobei “minimaler Login-Level” entspricht:

    • pwd_admin_e
    • pwd_super_user_e
    • pwd_user_e
    • pwd_operator_e

    Wenn ich zum Beispiel protection_level auf pwd_user_e setze, kann ich die Seite nur betreten, wenn ich mit dem Passwort von “User”, “SuperUser” oder “Admin” angemeldet bin. Andernfalls erscheint ein Popup, das Sie darüber informiert, dass Sie keinen Zugriff auf die Seite haben. Denken Sie daran, Ihre Änderungen auf der Seite zu speichern.

    Wenn im Seitenkonstruktor keine Zugriffsebene angegeben ist, hat die Seite pwd_operator_e als Standardzugriffsebene, d.h. die Seite ist für alle bereitgestellten Benutzerebenen und auch für jeden Benutzer zugänglich, der sich nicht am System angemeldet hat.

    HINWEIS: Jeder Benutzer darf nur sein eigenes Passwort ändern. Das heißt, der Benutzer “User” kann nur sein Passwort ändern und der Benutzer “Admin” kann das Passwort “User” nicht ändern.

    Die change password taste ist deaktiviert, bis sich ein Benutzer anmeldet.

    Die Anmeldung verfällt erst, wenn das Panel neu gestartet wird oder wenn Sie sich nicht explizit durch Drücken der entsprechenden Schaltfläche abgemeldet haben. Sie können jedoch eine maximale Leerlaufzeit festlegen, nach der die Anmeldesitzung abläuft. Direkt aus dem “System Editor” ist es möglich, die Zeit (Pwd Timeout [s]) und die Zielseite für den Fall festzulegen, dass die Sitzung abläuft (Pwd Logout Page).
