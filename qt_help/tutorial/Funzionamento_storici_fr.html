Fonctionnement de l'historique

Le panneau stocke toutes les grandeurs relatives aux variables Crosstable qui ont dans le champ “Update” mis à la valeur:

  • S: : La variable est toujours mise à jour et stockée lentement avec un temps “Slow Log Period” (paramètre System Editor → System);

  • F: comme “S” mais rapidement et avec un temps “Fast Log Period” (paramètre System Editor → System);

  • V: la variable n'est acquise et historisée que lorsqu'il y a un changement;

  • X: La variable est historisée lorsqu'elle est appelée par la fonction logShot().

Le mécanisme de log peut être démarré, arrêté et forcé à volonté en utilisant les fonctions logStart();, logStop(); et logShot();. La fonction logReset(); supprime tous les logs sur l'appareil. Ces fonctions seront mises dans le code désiré comme indiqué dans le chapitre “Réalisation d'une automatisation”.

L'historique généré est journalier et unique pour toutes les quantités activées pour cette fonction Cependant. Un nouveau fichier est généré à logStart(); (un nouveau fichier est automatiquement créé à 00h00).

Pour afficher la page d'historique, vous pouvez utiliser un ATCMbutton en mode changement de page et entrer dans la propriété “pageName” le nom de la page de bibliothèque “store”.

Le bouton permet le filtrage dans le temps des historiens tandis que le bouton , qui sera activé lors de l'insertion d'une clé USB, vous permettra de télécharger des données sur une clé USB.


Il est possible de définir une série d'“historiques partiels” pour lesquels l'affichage du panneau est disponible.

Le nouveau fichier doit être téléchargé dans l'appareil via l'interface Web web, voir chapitre “Fonctionnement de l'interface Web”. Si vous le souhaitez, vous pouvez spécifier un sous-ensemble des variables historisées en écrivant un fichier de configuration (store1.csv) de cette manière:

date
time
TPLC_IN_3
TPLC_IN_AN_2

Pour afficher la page d'historique, sélectionnez MENU → DATA → STORE, vous pouvez aussi utiliser un ATCMbutton en mode changement de page et entrer dans la propriété “pageName” le mot “nom de fichier” sans extension. Où “nom de fichier” sera un fichier.csv qui devrait commencer par “store”.
