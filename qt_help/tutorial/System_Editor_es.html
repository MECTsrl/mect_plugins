Editor de sistema

Con el System Editor es posible modificar algunas configuraciones para la interfaz gráfica y los parámetros de comunicación entre el panel y los nodos remotos.

Para abrir el “System Editor” seleccione el icono MectSuite:


Ecco come si presenta il “System Editor”:

En la parte superior de la pestaña “System” è se encuentra el botón para guardar los ajustes. A continuación se muestran algunas pestañas: System, SERIAL PORT 0 etc…. que pueden ser editadas o no dependiendo de los puertos disponibles en el producto.

Aquí están las características del “System Editor”:

  • System: se fijan todas las características generales del sistema:

    • Pwd Logout Page: Página que se muestra cuando finaliza la sesión. Si el campo está vacío, la pantalla actual permanece.

    • Pwd Timeout [s]: Indica después de cuántos segundos de inactividad expira la sesión. Con el valor “0” la sesión nunca expira.

    • Screen Saver [s]: Indica al cabo de cuántos segundos comienza a funcionar el modo “Salvapantallas”. Con el valor “0” se desactiva el ajuste.

    • Slow Log Period [s]: Indica cuántos segundos se desea memorizar las variables del tipo “S” (definidas en el campo “Update” del “Crosstable Editor”).

    • Fast Log Period [s]: Indica cuántos segundos se desea memorizar las variables del tipo “F” (definidas en el campo “Update” del “Crosstable Editor”). El número debe ser menor que el campo “Slow Log Period”.

    • Max Log Space [Mb]: Cantidad de espacio dedicado a guardar variables historizadas.

    • Trend Window [s]: Contiene el valor de la ventana de tiempo de visualización de trend. El número debe ser al menos tres veces el campo “Fast Log Period”.

    • Retries: indica el número de errores permitidos a un dispositivo antes de reducir la tasa de interrogación.

    • Blacklist: Tasa de interrogación reducida si se excede el número establecido en el campo “Retries”.

    • Start Page: Nombre de la página de la pantalla que desea visualizar en primer lugar. En Qt Creator debe estar presente una página con este nombre.

    • Home Page: Nombre de la página que desea asociar a la “Home Page”. En Qt Creator debe estar presente una página con este nombre.

    • Buzzer Touch: Indica si se debe asociar una señal acústica con cada evento de presión en la pantalla táctil.

    • Buzzer Alarm: Indica si la ocurrencia de una condición de alarma debe estar asociada a una señal acústica.

    • Read Period 1 [ms]: indica con qué frecuencia se leen las variables con valor “1” en el campo “Priority” de la Crosstable.

    • Read Period 2 [ms]: indica con qué frecuencia se leen las variables con valor “2” en el campo “Priority” de la Crosstable.

    • Read Period 3 [ms]: indica con qué frecuencia se leen las variables con valor “3” en el campo “Priority” de la Crosstable.

      El campo “Priority” no se tiene en cuenta para las variables con protocolo PLC.

  • SERIAL PORT 0 / 1 / 2 / 3: se ajustan todos los parámetros de comunicación del dispositivo serie.

    En el campo “Max Block Size” se puede definir el número máximo de variables dentro de un bloque.

    ATENCIÓN: el reordenamiento de los bloques se realiza después de guardar la Crosstable. Los parámetros internos de la serie no deben modificarse, ya que de lo contrario el dispositivo no funcionará correctamente.

  • TCP IP PORT: se ajustan los parámetros para la comunicación TCP IP con un dispositivo remoto:

    • Timeout [ms]: tiempo máximo de espera para que el Master reciba una respuesta del Slave.

    • Silence [ms]: frecuencia de peticiones consecutivas de un Master a un Slave.

    • Max Block Size [1…64]: número máximo de variables dentro de un bloque. ATENCIÓN: el reordenamiento de los bloques se realiza después de guardar la Crosstable.

  • CANOPEN 0 / 1: Ajustes de velocidad de transmisión del protocolo CANopen.
